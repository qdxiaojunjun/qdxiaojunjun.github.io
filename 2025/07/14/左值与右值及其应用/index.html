左值与右值及其应用

一、左值与右值的概念

1. 定义

  • 左值：可放置在赋值运算符左侧的值，具有持久存储位置的对象
  • 右值：不具有固定存储位置的临时值，表达式求值后立即销毁
  • C++标准定义：表达式求值后具有身份（identity）和存储位置的是glvalue（泛左值），可以被移动的是rvalue（右值）

2. 左值特点

  • 持久性：生命周期超出当前表达式
  • 可寻址：可用&获取内存地址
  • 可赋值：可放在赋值表达式左侧
  • 具名性：通常有标识符名称

3. 右值特点

  • 临时性：在表达式结束即销毁
  • 不可寻址：无法获取内存地址
  • 不可赋值：不能放在赋值表达式左侧
  • 匿名性：通常没有标识符名称

4. 值类别划分

    graph TD
    A[表达式] 
    A --> B[泛左值 glvalue]
    A --> C[右值 rvalue]
    B --> D[左值 lvalue]
    B --> E[将亡值 xvalue]
    C --> E[将亡值 xvalue]
    C --> F[纯右值 prvalue]
类别 名称 特征 示例
lvalue 左值 有标识&持久 具名变量int x = 5;
xvalue 将亡值 有标识但可被移动 std::move(x)
prvalue 纯右值 无标识&临时 42, func()
glvalue 泛左值 有标识 lvalue + xvalue
rvalue 右值 可被移动 xvalue + prvalue

二、左值引用与右值引用

1. 引用类型对比

特性 左值引用 (T&) 常量左值引用 (const T&) 右值引用 (T&&)
符号 & const & &&
可绑定左值 ✅ ✅ ❌
可绑定右值 ❌ ✅ ✅
可修改对象 ✅ ❌ ✅
主要用途 避免拷贝 通用访问 资源转移

2. 左值引用 (T&)

基本概念：只能绑定到左值的引用，提供对现有对象的别名访问

实现原理：在编译层面维护指针语义，运行时零开销

3. 右值引用 (T&&)

基本概念：只能绑定到右值的引用，用于移动语义

关键特征：

  • 延长临时对象的生命周期
  • 支持移动语义避免深拷贝
  • 实现完美转发

4. 常量左值引用 (const T&)

特殊属性：可同时绑定左值和右值的只读引用

优化价值：避免不必要的对象拷贝

注意： 一个右值引用变量，本身是一个左值！！！！！！！
